安装 vs2005 sp1

可怜我的笔记本C盘一共也只有9个G，惨！惨！惨！

以下内容转载自：http://www.cnblogs.com/index/archive/2009/02/18/673692.html

恐怖的vs2005 sp1安装过程

今天刚把这个400多M的sp1从公司拷回来, 算计着这样可以省掉不少下载时间.
可是, 可是, 竟然没有想到, 安装过程是如此的恐怖.

首先, 我的C盘空间明显的感到害羞, 只有1.5G的剩余空间了, 赶快压缩一下, 转移一下虚拟内存, 腾出了5个G的空间来伺候这位爷.
(刚看到微软官方说明, 最好有6个G的剩余空间, nnd, 别吓我, 我已经装了一晚上了)

终于, 点击安装的exe, 漫长的等待开始了…
经过长达30分钟的第一次收集信息, 然后出现几个提示让你选Yes, 然后又是长达n分钟的收集信息
这个n我还没有统计出来, 因为我大概晚上8点开始装这个sp1, 一直到现在(晚上11点)仍然在收集信息, 状态条大概过去了80%
(打完这行字终于出现了一个剩余时间, 10分钟)
这3个小时中, cpu的占用率一直是100%, 你啥也干不了, 我这篇post还是用MM的本本写的.

还有很大的可能安装不成功, 因为据说这个sp1实在太难伺候了…

sp1的安装过程到底干了什么? 以下内容引用自http://blog.joycode.com/jiangsheng

OK，我们来看看Visual Studio 2005 SP1安装的时候到底做了什么。


  • 解压到%temp%\临时目录（对于我的Visual Studio 2005 Team System，这需要C:\Documents and Settings\Administrator\Local Settings\Temp下面1个多GB的空间）

  • 把自己复制到%temp%下（400多MB）

  • 把自己复制到%windows%\Installer下（C:\Windows Installer下400多MB）

  • 解压到%windows%\Installer\PatchCache（1个多GB）


安装过程中还会生成临时文件（C:\Config.msi和%temp%下面的日志文件）。也就是说，要成功安装Visual Studio 2005 SP1的话，所需空闲空间大大超出成功安装Visual Studio 2005所需的空间。在SP1的下载页面，列出的空间要求是6.2GB（这还是在没有安装多个Visual Studio 产品的情况下）。

本站使用的 WordPress 插件及使用方法(备忘)

2009/11/11 更新

wordpress 的插件很丰富，用法各有不同，所以还是做一下记录，方便下次安装和升级。

日志：


  • No Self Pings: 禁止链接站内文章时发送 pingback 和 traceback

  • WordPress Related Posts : 根据 tag 显示相关文章，需要在模板适当位置插入代码：<?php wp_related_posts(); ?>为了美观，可能还需要修改css代码

  • WP-CodeBox: 使用GeShi, 支持语言多。但是我发现使用这类插件时，在编辑器的HTML和可视化两种模式之间切换时， <pre> 中间的一些标签会被过滤掉，如 line, 虽然结果正常，但是感觉还是不太好。该插件的用法如下：

    <pre lang=”cpp” line=”1” file=”code.txt” colla=”+”>
    int main()
    {
    printf(“hello WP-CodeBox\n”);
    return 0;
    }
    </pre>



评论：

  • WP-RecentComments: 显示最新评论内容，放置在侧栏，支持头像

  • Akismet: 官方的防垃圾评论插件


性能：

  • WP Super Cache: 页面缓存，增加访问速度

  • Delete-Revision: 删除多余的修订版本，优化数据库


搜索引擎优化：

  • Google Analytics for WordPress: 方便增加 Google 分析代码

  • Google XML Sitemaps with qTranslate Support: 生成 Sitemap

  • Redirection: 重定向页面, 文章修改后地址后，可以将老的地址定向到新地址

  • WP-ShortStat: 访问统计
