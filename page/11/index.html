本站使用的 WordPress 插件及使用方法(备忘)

2009/11/11 更新

wordpress 的插件很丰富，用法各有不同，所以还是做一下记录，方便下次安装和升级。

日志：


  • No Self Pings: 禁止链接站内文章时发送 pingback 和 traceback

  • WordPress Related Posts : 根据 tag 显示相关文章，需要在模板适当位置插入代码：<?php wp_related_posts(); ?>为了美观，可能还需要修改css代码

  • WP-CodeBox: 使用GeShi, 支持语言多。但是我发现使用这类插件时，在编辑器的HTML和可视化两种模式之间切换时， <pre> 中间的一些标签会被过滤掉，如 line, 虽然结果正常，但是感觉还是不太好。该插件的用法如下：

    <pre lang=”cpp” line=”1” file=”code.txt” colla=”+”>
    int main()
    {
    printf(“hello WP-CodeBox\n”);
    return 0;
    }
    </pre>



评论：

  • WP-RecentComments: 显示最新评论内容，放置在侧栏，支持头像

  • Akismet: 官方的防垃圾评论插件


性能：

  • WP Super Cache: 页面缓存，增加访问速度

  • Delete-Revision: 删除多余的修订版本，优化数据库


搜索引擎优化：

  • Google Analytics for WordPress: 方便增加 Google 分析代码

  • Google XML Sitemaps with qTranslate Support: 生成 Sitemap

  • Redirection: 重定向页面, 文章修改后地址后，可以将老的地址定向到新地址

  • WP-ShortStat: 访问统计

此时我有个邪恶的想法，等 Google wave 正式发布以后，腾讯，百度啊赶紧上，整个山寨的出来，至少放在国内不用担心被墙。

GRUB4DOS 使用备忘

由于安装顺序的问题，比如，在已有 linux 的硬盘上安装 XP会覆盖 MBR，导致linux无法启动，又或者安装 linux 时没有把 grub 安装到 MBR，这时可以利用 grub4dos 来引导 linux。

GRUB4DOS 启动方式有很多种，参考《Grub4dos安装和启动》，这里是用windows 的启动管理器来启动。

从 GRUB4DOS 主页下载最新版，这里下载的是0.4.4版，解压并复制里面的 grldr 文件到C盘根目录，再修改 C:\boot.ini文件，增加一行:

c:\grldr="Start GRUB4DOS"
机子启动后，选择Start GRUB4DOS，自动会转到从盘的GRUB，这样双系统就可以正常使用。 如果想用linux 的 grub来引导，需要修改grub的menu.lst文件，增加windows引导选项，再用 grub-install 命令重新安装 grub 到MBR，命令类似于 
sudo grub-install /dev/sda

具体使用方法见下面两篇文章：

  • LiveSpace的搬家利器：Live Space Mover
