色情图片识别技术

图像检测进程从待检图像队列中获取图像 数 据 ,首先将图像尺寸做归一化处理, 把宽度大于 200 像素的图像大小调成为宽度为 200 像素的图像。 然后采用区域肤色分割的方法将肤 色 区 域 和 非 肤 色 区 域 分 离 出 来 ,将 非 肤 色 区 域 添 黑 。在 对 肤 色 区域关系进行分析后去除干扰肤色区域(类肤色背景) , 提取区域的数量 、 大 小 、 位 置 、 比 率 等 特 征 送 入 已 经 训 练 好 的 S VM 分 类 器 中 进 行分类。当图像被检为色情图像后送入人脸检测器中检测为非人脸时, 便确定为色情图像。

以上摘自郑州金惠计算机系统工程有限公司的《绿色上网过滤软件产品一年使用权及相关服务采购项目》,好像叫谈判响应书吧。网上流传中…

里面提到不少技术，很值得我们民工学习，人家的软件一下子能卖4000多万，还只是一年服务的。

学车记录

6.1 报名

6.3 拍照、体检[67￥]，车管所理论考报名[260￥]，等开班中…

6.3 驾校凑不齐人开班，直接免读，条件是先把驾驶培训费交了，so 培训费 [2600￥]，IC卡工本费[35￥]

7.7 交了考务费360，下周三理论考试

7.15 理科考试，90分勉强通过，比起同行的牛人们的98、99、100高分，实在汗颜。错了的10道题中大部分没有见过或者只瞥过一次，所以答题不确定。考试结束后问问别人，原来他们都做过这些题。看来如果要保险一点过理论考，还是需要猛做模拟题。

9.4 终于开始学车了。今天第一天，教了下怎么换档和转方向盘。还是有点搞不清楚离合器到底是怎么工作的。

9.5 上午练习直线，也就是起步、停车、倒车。开始比较慌，慢慢适应了。下午学习四个项目：连续障碍（大饼）、曲线、直角转弯、单边桥。曲线和直角转弯比较简单，一般不会出错。

9.6 上午练习昨天新教的四个项目，下午学了下侧方停车，比较简单。然后接着练那四个项目。还有好几个项目没学，马上又要考试，时间安排的实在太紧了。现在我们这组又增加了三个人，一共6个。希望接下来这几天我们这几个人时间能错开点。

9.7 上午去练了两个钟头，试了下跑外圈，也就是上坡起步，定点停车，起伏路这些，这次是在师姐的陪同下试，下次教练应该会教一下的。

9.8 定点停车还是比较难，车头停在15cm宽的线内，不太可能吧，另外就是倒桩，至今没有练过。晚上差点轧了一窝小狗，万幸万幸。

9.9 目前存在的问题

倒桩还没练


其他项目偶尔会出错

  1. 上车后要调整好座位，还有人坐端正，不歪头。这些很影响视线

  2. 换档、拉手刹或其他操作不要去看，目光要正视前方

  3. 停车时，先踩住离合器和脚刹，拉手刹，档位拨到0档，再松离合器和脚刹

养成习惯！！

9.10 晚上跑了五圈，还可以。明天下午开始学倒桩

9.11 - 9.13 期间去考场试了一下场地，在驾校的红外线场地体验了一把倒桩考试。这些都是收费的，现在每人交了200，几乎可以肯定是不够的。其他时间就是练习练习再练习，脑袋都昏了。明天去考场练习倒桩了，后天考试

9.14-9.15 14号晚上 5:00 - 9:00 练习倒桩，场地够破的，不过主要目的是熟悉车，因为这也是第二天考试要用的车。 15号凌晨4点多练习综合项目，师傅模拟考试，每次让我们练习四个项目，有问题的地方再纠正一下，骂还是照样要骂的。7点吃过早饭以后，排队等候倒桩考试， 几乎等了2个小时，郁闷死。还好考试时没出岔子，虽然还是有点危险。考完倒桩之后直接考综合项目：侧方停车、定点停车和上坡起步、直角转弯、单边桥，这次运气也比较好，扣了20分之后还是通过了，没用上补考机会。 XD

费用统计：两次场地费，住宿费[标间160￥]、14号的晚饭、其他费用。个人花费大概220左右 。目前总花费3742￥

Gmail 发送带可执行文件的附件

Update: 试了一下，Winrar 压缩包带exe文件，可以发送。难道我记错了?

用Gmail发送附件时，Gmail 会根据文件后缀名判断，不让用户发送 exe 后缀的文件，但总是有需要发送正常的exe文件，怎么办呢？


  • 修改文件后缀名，让收件人修改回来。

  • 用7-zip 压缩再发送。winrar 和 zip 压缩不管用，Gmail 还是能看到压缩包内的文件名。

  • 分卷压缩，我用rar成功了

我知道的就这几种办法…

[zz]VC程序重定向printf到console窗口

本方法简单易用，适用于VC6、VC.NET2003

假定你创建的VC应用程序叫做ImageFilter，那么只需添加如下代码即可:

[1] 打开ImageFilter.cpp文件，增加

#include   
#include

[2] 在该文件的’CImageFilterApp theApp;’后面增加一个函数:

void InitConsoleWindow(void)
{
    int hCrt;
    FILE *hf;
    AllocConsole();
    hCrt = _open_osfhandle((long)GetStdHandle(STD_OUTPUT_HANDLE),
                    　　　　　　　 _O_TEXT );
    hf = _fdopen( hCrt, "w" );
    *stdout = *hf;
    setvbuf( stdout, NULL, _IONBF, 0 );

    // test code ...
    printf("InitConsoleWindow OK!\n\n");
}

[3] 在函数’BOOL CImageFilterApp::InitInstance()’中的所有代码前面调用该函数:

InitConsoleWindow();

好了大功告成! 后面你就可以使用printf输出运行信息了 :)
