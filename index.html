adb 网络调试

通过网络用 adb 调试应用的步骤：


  1. 保证你的 Android 设备和开发计算机在同一局域网内。

  2. 用 USB 线连接 Android 设备到开发计算机，确保 Android 设备的 USB 调试功能已经打开

  3. cmd 中输入 adb tcpip 5555

  4. cmd 中输入 adb connect <DEVICE_IP_ADDRESS>:5555

  5. 断开 USB 连接，开始正常的 adb 调试。

  6. 要回到 USB 调试时，输入 adb -s <DEVICE_IP_ADDRESS>:5555 usb


注：

  1. 不需要root

  2. <DEVICE_IP_ADDRESS> 即 Android 设备在局域网中的 IP 地址，可在系统设置中的“网络”->“高级” 中查看。

恒宝长大ing

以前没断奶带身边的时候，因为天天见，总是看不见哼宝的长大，总是那个抱在手里软软的带着奶味的宝宝。

就是到后面越来越抱不住，在你怀里老扭捏着，到后面不用给他很痛苦地断奶，因为他自己已经不想待在我的怀里喝奶了。

断奶之后一直是奶奶很辛苦带他，陪他出去玩出去逛，这个是宁波养成的习惯，哪天不带出去逛，就哼唧哼唧闹腾甚至哭。

到后面开始会喊爸爸妈妈，很对不起，没带在身边，也没亲耳听到，见证那激动人心的时刻。后来哼爸回去录了视频，视频里奶声奶气喊得确是爸爸，有点点小伤心，却装作不在意。

过年回去的时候，还不大会走路，爬去厨房或者房间找奶奶的那个利索，如果没看到你是不会相信的。哼宝不知道是不是因为爬的比较早相对走路就晚了，不知道这个存不存在这样的逻辑不，过只能安慰自己说走路晚的宝宝以后福气会多一点。

清明回去第一次上坟，哼妈抱了一小段哼宝走很平坦的水泥路，不过却挺陡的，累得够呛的，后面一直奶奶抱着，奶奶真是辛苦啊。到了公墓台阶那里，哼宝跪着爬阶梯，被哼姑姑称赞很有孝心，贼逗。

那时候还是只会走一点点，就几步而已，当快扑到你怀里的时候整个人就往前倾，如果不去抱一准摔倒。而当你强制把他放在那边靠墙，他不愿意走路的时候，他就哭闹给你看，甚至坐在地上。

而且还在不久前把爷爷的手机摔坏了，没轻没重的娃加上特别宠孙子的爷爷，能不摔坏手机吗？看见你打手机，就要抢过来放在耳边，却不和你说话，也不知道电话那段多么期待他喊声爸爸妈妈，奶奶在一边叫，他却很淡定抱着电话。

清明回去后，哼爸给爷爷买了新手机，翻盖打开后会有“叮咚”提示音，然后按数字键会报数。听说哼宝去翻爷爷的腰包，没找到手机，嘴巴一扁就要哭了，爷爷赶忙给他新手机。一翻开，叮咚，小家伙顿时乐呵了，再按数字有报数，更加笑得见眉不见眼。从此小朋友又多了一个不贵的手机，超贵的玩具。

最近哼姑姑微信过来，发了好多哼宝的照片，据说小家伙已经不用大人扶着站好，而且走得比那次回去更好了，小家伙开始向直立行走迈进一大步，哼爸很骄傲也很得意。相信，奔跑指日可待。

小家伙越长越大，站起来也快有饭桌那么高了，有时候觉得母爱泛滥之后，有孩子慢慢长大自己慢慢老了的感觉。

KB2640696

之前写的程序有涉及到 ado 访问 access 数据库，在其他计算机上运行时返回0x80004002(E_NOINTERFACE)错误，Google之后发现原因

Windows 7 Sp1 和 Windows Server 2008 R2 SP1 采用了新的 ADO IIDs，导致所编译的程序在之前版本的操作系统上无法访问access数据库，比如WinXP，Win7。


解决方案在这里：http://support.microsoft.com/kb/2640696

安装对应版本的补丁（官方下载需正版验证），然后将代码中的

#import msado15.dll

改为

#import msado28.tlb

重新编译，搞定
